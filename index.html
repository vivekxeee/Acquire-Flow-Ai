01 // AGENT PIPELINE — SEQUENTIAL ORCHESTRATION
AGENT A
The Researcher
LEAD INTELLIGENCE SPECIALIST
Scrapes LinkedIn company pages and the open web to build a structured Lead Dossier. Detects funding events, tech stack, hiring signals, and key decision makers.
▸ LinkedIn Scraper (ProxyCurl)
▸ Web Research (DuckDuckGo)
▸ Apollo Lead Enrichment
→
AGENT B
The Strategist
GTM PAIN-POINT ANALYST
Ingests the Dossier and identifies the top 3 pain points grounded in real signals. Maps each pain to a specific product value prop. Outputs a "Strike Brief" with a hook sentence.
▸ Pure Reasoning (No Tools)
▸ Context: Dossier from Agent A
▸ Output: Strike Brief (MD)
→
AGENT C
The Copywriter
HYPER-PERSONALIZED OUTREACH
Writes a 3-email cold outreach sequence using the Strike Brief. Email 1: Hook. Email 2: Value + Case Study. Email 3: Pattern-interrupt breakup. Zero templates. Every line earned.
▸ Context: Dossier + Strike Brief
▸ Forbidden phrases list enforced
▸ Output: 3 Emails (sub/body)
→
AGENT D
The Closer
RAG-POWERED OBJECTION HANDLER
When a prospect replies with an objection, retrieves the top winning responses from the vector playbook, crafts a personalized reframe, sends reply, and logs to CRM.
▸ RAG Playbook (ChromaDB)
▸ Email Sender (SendGrid)
▸ CRM Updater (HubSpot)
02 // RAG ENGINE — OBJECTION INTELLIGENCE
RAG Playbook Engine
RETRIEVAL-AUGMENTED GENERATION
A local ChromaDB vector store ingests your winning sales responses, email threads, deal post-mortems, and objection scripts. When a prospect objects, Agent D retrieves the 4 most relevant playbook entries using MMR search and augments the LLM prompt with real-world context — no hallucinated responses.
RETRIEVAL PIPELINE
01 Prospect reply ingested as raw text
02 Embed objection via nomic-embed-text (Ollama)
03 MMR search in ChromaDB (k=4, fetch_k=20)
04 Top 4 playbook hits injected into Closer's prompt
05 Llama 3/Mistral generates personalized reply
06 Reply sent + outcome logged to CRM
TECH STACK
Vector DBChromaDB (local)
Embeddingsnomic-embed-text
LLM (Privacy)Llama 3 / Mistral
LLM RuntimeOllama (local)
Search TypeMMR Retrieval
Chunk Size500 tokens
